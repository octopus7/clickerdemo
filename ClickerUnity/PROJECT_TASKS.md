## 규칙
- 작업 시작 시 태스크를 `대기`에서 `진행 중`으로 이동
- 관련 변경의 컴파일/테스트 통과 후에만 `완료`로 이동
- 태스크 ID는 고정 유지 (`T-001`, `T-002`, ...)
- 사소한 수정은 `사소한 수정`에 태스크 번호 없이 간단히 기록
- 기존 태스크의 오류 재수정은 `사소한 수정`에 기록하지 않음
- 기존 태스크 오류 수정은 중요도를 판단해 필요 시 해당 태스크에 보완기록으로 남김
